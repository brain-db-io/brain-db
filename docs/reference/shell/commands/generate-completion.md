# `brain generate-completion`

Emit a shell-completion script to stdout for one of the supported
shells. Reach for this once per machine — pipe the output into the
location your shell auto-loads completions from, and tab-completion
covers every global flag and every subcommand's flags from then on.

```
brain generate-completion <SHELL>
```

`<SHELL>` is one of: `bash`, `zsh`, `fish`, `powershell`, `elvish`.

The script is generated by `clap_complete` against the full `Cli`
tree, so it reflects whatever the binary you ran knows about — if
a future release adds a new subcommand or flag, regenerating picks
it up. There is **no runtime dependency** on the `brain` binary
once the script is installed; completions are pure shell code.

---

## What gets completed

| Position | What you get |
|---|---|
| Subcommand slot | Every verb (`encode`, `recall`, `agent`, `config`, `info`, `shell`, …). |
| Flag slot (after a subcommand) | Every flag for that subcommand, plus every global (`--server`, `--agent`, `-o`, `--color`, `--hyperlinks`, `--timeout`, `--token`). |
| Enum-flag value slot | Restricted to the variants (`--kind episodic\|semantic\|consolidated`, `--mode soft\|hard`, `--output auto\|table\|wide\|json\|ndjson\|yaml`, `--color auto\|always\|never`, …). |
| Subcommand-of-subcommand slot | Subcommands like `agent list`, `config set`, `txn begin`. |

Dynamic completions (named-agent lookup from the config file,
context-id history) are **not** offered — the generated script is
static. If you want `--agent <TAB>` to expand to your configured
names, that's an in-REPL feature only (see
[`../repl-meta.md`](../repl-meta.md)).

## Install by shell

### bash

```bash
# System-wide
sudo brain generate-completion bash > /etc/bash_completion.d/brain

# Per-user
brain generate-completion bash > ~/.local/share/bash-completion/completions/brain
```

Reload with `source ~/.bashrc` or start a new shell.

### zsh

The first writable directory in `$fpath` is the conventional drop
location:

```zsh
brain generate-completion zsh > "${fpath[1]}/_brain"

# Or, if you keep your own dir:
mkdir -p ~/.zsh/completions
brain generate-completion zsh > ~/.zsh/completions/_brain
# and ensure ~/.zsh/completions is in $fpath BEFORE `compinit`
```

Run `compinit` (or restart) to pick it up. The file must be named
`_brain` for zsh to find it.

### fish

```fish
brain generate-completion fish > ~/.config/fish/completions/brain.fish
```

Fish auto-loads on next prompt; no restart needed.

### powershell

```powershell
brain generate-completion powershell | Out-String | Invoke-Expression

# To persist, append to your profile:
brain generate-completion powershell >> $PROFILE
```

`$PROFILE` is usually `~/Documents/PowerShell/Microsoft.PowerShell_profile.ps1`.

### elvish

```elvish
brain generate-completion elvish > ~/.elvish/lib/brain-completions.elv
echo 'use brain-completions' >> ~/.elvish/rc.elv
```

## Examples

```bash
# One-liner: install for the current shell only
case "$(basename "$SHELL")" in
  bash) brain generate-completion bash > ~/.local/share/bash-completion/completions/brain ;;
  zsh)  brain generate-completion zsh  > "${fpath[1]}/_brain" ;;
  fish) brain generate-completion fish > ~/.config/fish/completions/brain.fish ;;
esac

# Regenerate after upgrading the binary
brain generate-completion zsh > "${fpath[1]}/_brain" && exec zsh

# Inspect what the script offers without installing
brain generate-completion bash | less

# CI smoke — confirm the binary can at least emit a script
brain generate-completion bash | wc -l
```

## Errors

| Trigger | Outcome |
|---|---|
| Unknown shell name | clap parse error (exit 2); the help text lists the allowed values. |
| stdout is full / closed pipe | I/O error (exit 1). |

The verb does not connect to a server, read config, or touch the
agent file — it's purely a code-generator over the `Cli` tree, so
it works on a fresh machine before any `brain config` has been run.
